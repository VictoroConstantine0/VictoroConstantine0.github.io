SQLMap语句

指令 作用
-d 连接服务器上的数据库，将sqlmap作为一个服务器上的客户端程序
-u 指定url进行注入，必须有变量名称
-p 只检查某个参数
-f footprint检查足迹指纹信息
–users 查看当前数据库的账号
–banner 数据库的banner信息， 查看目标系统版本类型的数据库
-l 从burp或者webscarab的代理日志文件中的url进行逐个检查是否有注入点
-s 站点地图
-m 有多个url合并成一个文本文件然后提交给sqlmap进行逐个检查
-r http和body提交给sqlmap进行请求
-g 用google进行请求然后利用正则表达式进行过滤
-c 用配置文件进行扫描
-l log.txt (在option的misc里面生成日志文件)
-u “xxxxxx” –force-ssl
-c sqlmap.conf 配置文件扫描
–param-del 变量分隔符
–data 数据段
–cookie cookie头web应用需要基于cookie的身份认证 检查cookie中的注入点 Set-Cookie /–drop-set-cookie
–user-agent= 指定agent eg:sqlmap/1.0-dev-xxxxx (http://sqlmap.org)
–random-agent 随机agent /user/share/sqlmap/txt/user-agents.txt 不是已知的user agent的时候服务器可能会drop掉请求
–host=”xxx” (level=5)
–referer (level>=3)
–headers=”” eg:–headers=”host:xxxxxx\nUser-Agent:xxxxx” User-Agent大小写敏感
–auth eg:sqlmap -u “xxxx” –auth-type Basic –auth-cred “user:pass”
–auth-cert/-file PEM格式的证书链文件(含有私钥的PEM格式证书文件)
–proxy eg:–proxy=”http://127.0.0.1:8087"
–proxy-cred eg:–proxy-cred=”name:pass”
–ignore-proxy 忽略系统代理设置，通常用于扫描本地网络目标 eg:sqlmap -u “xxx” -proxy=”htttp://127.0.0.1:8087” -f
–delay 每次请求之间的延迟时间，浮点数，单位为秒，默认无延迟
–timeout 请求超时时间，浮点数，默认为30秒
–retries 了解超时重试次数，默认为3次
–randomize 长度，类型与原始值保持一致的前提下，指定每次请求随机取值的参数名 -randomize=”id”
–scope 过滤日志内容，通过正则表达式筛选扫描对象 sqlmap -l burp.log –scope=”(www)?\target.(com | net | org)”
–safe-url/freq 检测和盲注阶段会产生大量的失败请求，服务器端可能销毁session每发送–safe-freq次注入请求后，发送一次正常请求
–skip-urlencode 跳过url编码，默写WEB服务器不遵守RFC标准编码，使用原始字符提交数据
–eval 请求前执行指定的python代码sqlmap -u “xxx” –eval=”import hashlib;hash=hashlib.md5(id).hexdigest()”
–predict-output 根据检测方法，比对返回值和统计表的内容，不断缩小检测范围，提高检测效率 与–threads参数不兼容
–keep-alive 与 –proxy参数不兼容 长连接避免重复建立链接的网络开销，但大量长连接会严重占用服务器资源
–null-connection 之获取相应页面的大小之，而非页面集体内容 通常用于盲注判断真/假，降低网络带宽消耗 与–text-only参数不兼容
–threads=7 最大并发线程盲注时每个线程获取一个字符，获取完成后线程结束建议不要超过10，否则可能影响站点的可用性与 –predict-output参数不兼容
-o 开启前3三个性能参数(除–threads参数)
–skip 排除指定的扫描参数 eg： –level=5 –skip=”id.user-agent”
–dbms=”xxxx” 指定数据库
–os=”xxxx” 指定操作系统
–invalid-bignum/-logical 通常sqlmap使用负值使参数值失效 id=13 ->id =-13 bignum用大叔使参数值失效 id=9999999 logical使用布尔判断使取值失效 id=13 AND 18=19
–no-cast 榨取数据时，sqlmap将所有结果转换为字符串，并用空格和替换NULL结果 老版本mysql数据库需要开启此开关
–no-escape (使用该参数将关闭sqlmap使用char()编码逃逸的方法替换字符串)
–prefix /–suffix 前缀/后缀 –suffix “AND” (‘abc’=’abc’) –>query= xxxx AND (‘abc’=’abc’) xxxx;
–tamper 混淆脚本，用于绕过应用层过滤，IPS，WAF eg：–tamper=”tamper/between.py,tamper/randomcase.py,tamper/space2comment.py”-v 3
-dns-domain 攻击者控制了某DNS服务器，使用此功能可以提高榨取速度 eg： attacker.com
–second-order 在一个页面注入的结果，从另外一个页面显示出来 –second-order xxxx
–current-user 当前数据库用户
–current-db 当前数据库
–hostname 主机名
–users 所有用户的账号
–privileges -U –privileges -U username(CU当前帐号) 查询当前账户的权限
–roles 各种角色
–dbs 查询数据库
–table –exclude-sysdbs -D dvwa
-T users 指定表名
-D dvwa 指定数据库
-C user 指定列名
–common 常规
–schema –batch –exclude-sysdbs 元数据(使用默认选项)
–count 查询记录
–dump C(column), -T(table), -D(database), –start, –stop (具体第几条记录) –dump-all –exclude-sysdbs –sql-query “select * from users”
–common-columns/-tables Access系统无列信息
–file-read/-write eg：–file-write=”xxxxxx”Mysql postaresql 上传共享库并生成sys_exec()、sys_eval()两个UDF
–reg-read 读取注册表
–reg-add 增加注册表
–reg-del 删除注册表
–reg-key(键值名称)-value-data-type sqlmap.py -u= “xxx” –reg-add=”xxxxx” –reg-value=Test –reg-type=REG_SZ –reg-data=1”
-s sqlite会话文件保存位置
-t 记录流量文件保存位置
–charset 强制字符编码 –charset=GBK
–crawl 从起始位置爬站深度 –batch –crawl=3
–csv-del dump数据默认保存于”,”分割的csv文件中，指定其他分隔符 –csv-del=”;”
–dbms-cred 指定数据库账号
–flush-session 清空session
–force-ssl
–fresh-queries 忽略session查询结果
–hex dump非ASCII字符内容时，将其编码位16进制形式，收到后解码还原 sqlmap -u “xxx” –hex -v 3
–output-dir=/tmp
–parse-errors 分析和实现数据库内建报错信息 sqlmap.py -u “xxx” –parse-errors
–save 将命令保存成配置文件
-z 参数助记符 sqlmap –batch –random-agent –ignore-proxy –technique=BEU -u “xxx” \n sqlmap -z “bat,randoma,ign,tec=BEU” -u “xxxxx”
–answer 给问题的回答 –answers=”extending=N” –batch
–check-waf 检测WAF/IPS/IDS
–hpp 绕过WAF/IPS/IDS的有效方法，尤其对ASP/IIS和ASP.NET/IIS
–identify-waf 彻底的WAF/IPS/IDS检查 支持30多种产品
–mobile 模拟智能手机设备
–purge-output 清理output文件夹
–smart 当有大量检测目标时，只选择基于错误的检测结果
–wizard 向导
